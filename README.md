# Horeca Quest - Корпоративная игра

## База пользователей и восстановление пароля

### Запуск проекта

1. Установите зависимости:
```bash
npm install
```

2. Запустите сервер:
```bash
npm start
```

3. Откройте в браузере:
- Вход: http://localhost:3000/login.html
- Приложение: http://localhost:3000/app.html

### Учетные данные по умолчанию

**Режим Бога (Админ):**
- Логин: `admin`
- Пароль: `admin123`

### Возможности

**Режим Бога (god):**
- ⚙️ Шестеренка в хедере
- Добавление новинок (товаров)
- Удаление новинок
- Просмотр всех пользователей
- Удаление пользователей
- Назначение соадминов с Режимом Бога
- Управление базой данных

**Обычные пользователи (user):**
- 📌 Кнопка закладок в хедере
- Просмотр всех новинок
- Добавление новинок в закладки
- Удаление из закладок
- Просмотр своих закладок

### База данных

**users.json:**
- ID пользователя
- Имя пользователя
- Email
- ФИО
- Отдел
- Хешированный пароль
- Роль (god/user)
- Закладки
- Лучший результат
- Дата регистрации

**products.json:**
- ID товара
- Название
- Описание
- Категория
- Цена
- Иконка (эмодзи)
- Дата создания

### Восстановление пароля

1. Пользователь вводит email
2. Сервер генерирует токен восстановления
3. Токен выводится в консоль сервера (в реальном проекте отправляется на email)
4. Пользователь вводит токен и новый пароль
5. Пароль обновляется

### API Endpoints

**Авторизация:**
- POST /api/register - Регистрация
- POST /api/login - Вход
- POST /api/reset-password - Запрос восстановления
- POST /api/change-password - Смена пароля

**Пользователи (только god):**
- GET /api/admin/users - Список пользователей
- DELETE /api/admin/users/:id - Удаление пользователя
- POST /api/admin/promote - Назначение соадмина

**Новинки:**
- GET /api/products - Получить все новинки
- POST /api/products - Добавить новинку (только god)
- DELETE /api/products/:id - Удалить новинку (только god)

**Закладки:**
- POST /api/bookmarks - Добавить в закладки
- DELETE /api/bookmarks/:productId - Удалить из закладок
